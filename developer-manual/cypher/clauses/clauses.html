3.3. 절

읽기 절

이 부분은 데이터베이스에서 데이터를 읽는 절로 구성됩니다.

Cypher 쿼리 내의 데이터 흐름은 키-값 쌍(쿼리의 변수와 데이터베이스에서 파생 된 값 사이의 가능한 바인딩 집합)을 가진 정렬되지 않은 일련의 맵입니다. 이 집합은 쿼리의 후속 부분에 의해 구체화되고 보강됩니다.

절설명
MATCH데이터베이스에서 검색 할 패턴을 지정.
OPTIONAL MATCH데이터베이스에서 검색 할 패턴을 지정하며 패턴에서 빠진 부분에 대해서는 nulls를 사용.
WHEREMATCH 또는 OPTIONAL MATCH 절의 패턴에 제약 조건을 추가하거나 WITH 절의 결과를 필터링.
START레거시 인덱스를 통해 시작점을 발견함.
Aggregationcount(), sum(), avg(), max(), min(), collect() 등을 포함하는 집계 함수. DISTINCT 포함.
LOAD CSVCSV 파일에서 데이터를 가져올 때 사용.


쓰기 절

이 부분은 데이터베이스에 데이터를 쓰는 절로 구성됩니다.

절설명
CREATE노드 및 관계를 생성.
MERGE그래프에 패턴이 존재하는지 확인. 패턴이 이미 있는지 생성될 필요가 있는지 여부
SET노드의 레이블과 노드 및 관계의 속성을 업데이트.
DELETE그래프 요소를 삭제. - 노드, 관계 또는 경로
REMOVE노드와 관계에서 속성과 레이블을 제거.
FOREACH경로의 구성 요소든지 집계 결과든지 목록 내의 데이터를 업데이트.
CREATE UNIQUEMATCH와 CREATE의 혼합. 가능한 것을 매칭하고 빠진 것을 생성.
Importing CSV files with CypherLOAD CSV를 사용하여 CSV 파일에서 데이터를 가져 오는 방법.
PERIODIC COMMITPERIODIC COMMIT의 사용 방법 및 시기


일반 절

절설명
RETURN쿼리 결과 집합에 포함 할 내용을 정의.
ORDER BY출력을 특정 방식으로 정렬되도록 지정하는 RETURN 또는 WITH 뒤에 오는 하위 절.
LIMIT출력할 행 수를 제한.
SKIP어느 행부터 출력의 행에 포함할 것인지 정의.
WITH다음의 시작점 또는 기준으로 사용되도록 결과를 넘겨, 쿼리 파트가 함께 연결되게 함.
UNWIND목록을 일련의 행으로 확장.
UNION여러 쿼리의 결과를 결합.
CALL데이터베이스에 배포 된 프로시저를 호출.
